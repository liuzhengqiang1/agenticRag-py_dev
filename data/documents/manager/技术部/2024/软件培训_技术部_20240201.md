# 软件开发规范培训

## 第一章：代码规范

### 1.1 命名规范

变量和函数命名要遵循以下原则：

- 使用有意义的名称
- 遵循驼峰命名法
- 避免使用单字母变量（除循环变量外）
- 常量使用全大写加下划线

### 1.2 注释规范

代码注释要求：

1. 每个函数必须有文档注释
2. 复杂逻辑要添加行内注释
3. 注释要简洁明了
4. 及时更新注释内容

## 第二章：Git 使用规范

### 2.1 分支管理

分支命名规范：

- main：主分支
- develop：开发分支
- feature/xxx：功能分支
- bugfix/xxx：修复分支

### 2.2 提交规范

Commit 信息格式：

```
<type>: <subject>

<body>
```

type 类型：

- feat：新功能
- fix：修复bug
- docs：文档更新
- style：代码格式调整
- refactor：重构
- test：测试相关

## 第三章：测试规范

### 3.1 单元测试

单元测试要求：

- 覆盖率不低于80%
- 测试用例要包含正常和异常情况
- 使用 Mock 隔离外部依赖
- 测试代码要易于维护

### 3.2 集成测试

集成测试关注点：

1. 模块间接口调用
2. 数据流转正确性
3. 异常处理机制
4. 性能指标验证
