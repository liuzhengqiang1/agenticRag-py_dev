# 文档目录结构说明

## 目录组织规范

```
data/documents/
├── public/          # 公开文档（所有人可见）
│   └── {部门}/
│       └── {年份}/
│           └── {标题}_{作者}_{YYYYMMDD}.md
│
├── employee/        # 普通员工级别文档
│   └── {部门}/
│       └── {年份}/
│           └── {标题}_{作者}_{YYYYMMDD}.md
│
├── manager/         # 管理层级别文档
│   └── {部门}/
│       └── {年份}/
│           └── {标题}_{作者}_{YYYYMMDD}.md
│
└── executive/       # 高管级别文档
    └── {部门}/
        └── {年份}/
            └── {标题}_{作者}_{YYYYMMDD}.md
```

## 文件命名规范

格式：`{标题}_{作者/部门}_{YYYYMMDD}.md`

示例：

- `销售培训_张三_20240115.md`
- `软件培训_技术部_20240201.md`
- `员工手册_运营部_20240301.md`

## 权限级别说明

| 级别      | 说明     | 可访问范围         |
| --------- | -------- | ------------------ |
| public    | 公开文档 | 所有人             |
| employee  | 普通员工 | 员工、管理层、高管 |
| manager   | 管理层   | 管理层、高管       |
| executive | 高管     | 仅高管             |

## 元数据提取规则

系统会自动从文件路径和文件名中提取以下元数据：

1. **access_level**：从第一级目录提取（public/employee/manager/executive）
2. **department**：从第二级目录提取（如：销售部、技术部）
3. **year**：从第三级目录提取（如：2024）
4. **title**：从文件名第一部分提取（如：销售培训）
5. **author**：从文件名第二部分提取（如：张三）
6. **created_at**：从文件名第三部分提取（如：20240115 -> 2024-01-15）

## 使用流程

1. 将 PDF 文档通过 MinerU 转换为 Markdown
2. 按照命名规范重命名文件
3. 放入对应的目录（根据权限级别和部门）
4. 运行构建脚本：`python build_knowledge_es.py`
5. 系统会自动增量导入新文件或更新已变更的文件

## 注意事项

- 文件名中的日期必须是8位数字（YYYYMMDD）
- 文件名各部分用下划线分隔
- 标题和作者中不要包含下划线
- 确保文件编码为 UTF-8
- Markdown 文件建议使用标题结构（# ## ###）以获得更好的切分效果
